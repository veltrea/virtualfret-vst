# VirtualFret — 仕様書

ギターのフィンガーボード(指板)を再現したスクリーン MIDI キーボード。指板のクリックで単音、ラッチでコードを組み、ストラムゾーンのドラッグでストロークを再現し、MIDI ノートを下流の音源へ送る。音は出さない。

## 1. 背景

スクリーン MIDI キーボードは鍵盤型ばかりだが、ギタリストの頭の中にあるのは指板の配置(コードシェイプ、ポジション、チューニング)である。ギター音源(ギター向けサンプル音源・モデリング音源)へ MIDI を打ち込むとき、鍵盤 UI ではギター的なボイシングを直感的に作れない。

本プラグインは指板そのものを画面に置き、クリック・ドラッグでそのまま MIDI を送れるようにする。コードシェイプを指で覚えているギタリストが、その記憶のまま DAW に入力できる。

技術基盤は NoteNamer(`/Volumes/DISK/dev/notenamer-vst`)で確立した MIDI 送信 VST3 のパターンを全面再利用する。設計前に必ず `/Volumes/DISK/dev/knowledge/vst3_midi_out_plugin.md` を読むこと。

## 2. ゴール

- **6 / 7 / 8 / 9 弦**のギター指板を再現したスクリーン MIDI キーボード
- **クリックで単音発音**: セル(弦 × フレット)を押すと、その音程の MIDI ノートを出力に送る
- **チューニングの独立変更**: 見かけの指板デザインを変えずに、弦ごとに開放弦音程を半音単位で変更できる。ダウンチューニング・半音下げ・オープンチューニング等のプリセットを内蔵し、ユーザーが独自チューニングを追加保存できる
- **コードモード(ラッチ)**: 有効時はクリックした位置が点灯保持され、弦の違う場所を順にクリックしてコードを組める
- **ストラムゾーン**: 弦の上をピックでストロークするようなドラッグ操作で、時間差のあるストローク発音を再現する
- **コードプリセット内蔵**: 一般的なバレーコード・セブンスコード・ナインスコード等のシェイプライブラリから、ルート + タイプ選択でラッチを自動配置
- 設定・ラッチ状態は DAW プロジェクトと一緒に保存(プラグインステート)

## 3. やらないこと (Non-Goals)

- **音源機能** — 音は出さない。発音は下流のギター音源等が行う
- **ベンド / スライド / ビブラート / MPE** — アーティキュレーション表現は将来候補(v1 はノートオン/オフのみ)
- **アルペジエータ・ストラムパターンのシーケンス再生** — 将来候補。v1 のストラムは手操作のみ
- **タブ譜の読み込み・再生**
- **左利きモード・上下反転表示** — v1 後
- **カポの専用 UI** — チューニング(全弦一律オフセット)で代替可能。将来候補
- オーディオ処理、CC / ピッチベンド等の加工

## 5. MIDI 動作

- **スルー**: 入力 MIDI イベントはすべて無改変で出力へ(ノート・CC・全部)
- **生成ノートのマージ**: GUI 操作(クリック・ラッチ試聴・ストラム)から生成したノートを出力にマージする
- **スレッド設計**: NoteNamer パターンを踏襲 — GUI → lock-free FIFO(`AbstractFifo` + POD コマンド)→ `processBlock`。発音台帳 `sounding[16][128]` はオーディオスレッド専有。panic フラグで全消音
- **ノートオフ保証**(スタックノート防止):
  - エディタクローズ・再生停止時に発音中ノートを必ず解放(panic 経路)
  - **同一弦モノフォニック**: 同じ弦で新しい音を鳴らすとき、その弦の旧音を必ず先に off(実ギターの物理制約の再現でもある)
  - チューニング・弦数変更時は発音中ノートを全て off してから適用(off は鳴らしたときのノート番号で送る — 台帳がノート番号で持つので自然に正しい)
- **ベロシティ**: 単音クリック / ラッチ試聴 = 固定値(GUI 設定 1–127、既定 100)。ストラム = ドラッグ速度から算出(§9)
- **チャンネルモード**(GUI で切替):
  - **Single**(既定): 全ノート同一チャンネル(既定 1、1–16 で設定可)
  - **Per-string**: 弦ごとに別チャンネル(1 弦 = ch1、2 弦 = ch2、…)。弦別チャンネルを解釈するギター音源・MIDI ギター規格向け。ベースチャンネルをずらす設定を持つ
- **ストラムのタイミング精度**: ストロークの発音はブロック内**サンプルオフセット**で表現する。大きいオーディオバッファ(1024 サンプル等)でも、弦ごとの発音間隔が同一タイミングに潰れてはならない(GUI イベント時刻を保持し、processBlock 内でオフセットに換算・次ブロックへ持ち越し)

## 6. フレットボード(表示と単音演奏)

- **向き**: ヘッドが左、ボディ(ストラムゾーン)が右。横方向 = フレット、縦方向 = 弦
- **弦の並び**: 上 = 1 弦(最高音)、下 = 最低音弦(タブ譜と同じ向き)
- **フレット数**: 開放(ナット左の開放弦ゾーン)+ 1〜24 フレット
- **表示フレット数(ズーム)**: 幅に収めるフレット数を表示オプションで選択(12 / 15 / 18 / 21 / 24、既定 18)。データモデルは常に 24 フレットで、表示範囲外のポジションも発音はされる(ストラム・プリセット)— 描画されないだけ
- **ポジションマーク**: 3 / 5 / 7 / 9 / 12 / 15 / 17 / 19 / 21 / 24(12 と 24 はダブルドット)。フレット間隔・弦の太さ表現などの見た目は DESIGN.md(Stitch で作成)で詰める
- **弦数切替**: 6 / 7 / 8 / 9 をツールバーで切替。切替時はその弦数の既定チューニング(Standard)をロードし、ラッチをクリアし、発音中ノートを解放する
- **単音演奏**:
  - セルをマウスダウン = note on、マウスアップ = note off
  - 押下したまま同一弦上を横へドラッグ = フレット差し替え(旧 off → 新 on。ハンマリング/プリングの代替表現)
  - 押下したまま別の弦へ縦にドラッグ = 旧弦 off → 新弦 on(単純差し替え)
  - 開放弦ゾーンのクリック = 開放弦を発音
- **発音中表示**: 鳴っているセルを点灯(単音・ラッチ試聴・ストラム由来すべて)
- **入力 MIDI ハイライト**: 受信したノートと同じ音程を出せる**全候補ポジション**を薄く点灯(学習用途。設定で OFF 可)
- **ノート名表示**: セルに音名(C, C#, …)をオーバーレイ表示するオプション

## 7. チューニング

### データモデル

チューニング = **弦ごとの開放弦 MIDI ノート番号の列(低音弦 → 高音弦)**。セルの送出ノート = 開放弦ノート + フレット番号。

**見かけ不変の原則**: チューニングを変更しても指板の描画(フレット位置・ポジションマーク・弦)は一切変わらない。変わるのは各セルが送出するノート番号だけ。

### 弦ごとの独立編集

ヘッド部(指板左端)に各弦の開放弦音名を表示し、クリック / ▲▼ / ドラッグで**半音単位**に上下できる。これによりダウンチューニング・半音下げ・変則チューニングを弦ごとに自由に作れる。

### ユーザーチューニング

- 現在のチューニングに名前を付けて保存・削除できる
- 保存先はユーザーファイル(JUCE の userApplicationDataDirectory 配下 `VirtualFret/tunings.json`)。**全インスタンス・全プロジェクトで共有**(プラグインステートとは別)


(`strings` は低音弦 → 高音弦の開放弦 MIDI ノート番号。配列長 = 弦数)

## 8. コードモード(ラッチ)

ツールバーのトグルで ON / OFF(既定 OFF)。

- **ラッチ**: ON のときセルをクリックすると、その位置が点灯保持される
  - **1 弦 1 ポジション**: 同じ弦の別セルをクリックするとラッチ位置が置き換わる
  - 同じセルを再クリック → その弦のラッチを解除(= ミュート弦に戻る)
  - 開放弦ゾーンをクリック → 「開放(○)」をラッチ
  - **ラッチのない弦 = ミュート弦(×)**: ストラム時にスキップされる。コードダイアグラムの ×/○/フレットと同じ 3 状態
- **ラッチ試聴**: ラッチ・置換した瞬間にその音を短く発音する(設定で OFF 可、既定 ON)
- **クリア**: ツールバーのボタンで全ラッチ解除
- コードモード OFF への切替時は**ニュートラルへリセット**する: 全ラッチをクリアし、発音中のノートも全て解放する(トグル後に押さえ・点灯が残らない)
- ラッチ状態はプラグインステートに保存される

## 9. ストラムゾーン

指板の右側に常設する縦帯。ギターのボディ / ピックアップ部のメタファーで、各弦の延長線が通っている。

- **発音モデル**: ゾーン内でマウスダウン = ピックを構える(無音)。ドラッグでポインタが**弦の延長線を跨ぐたび**にその弦を発音する
  - コードモード ON: **ラッチ位置の音**(ミュート弦はスキップ、開放○は開放弦音)
  - コードモード OFF: 開放弦の音(現在のチューニングそのまま)
- **ストローク方向**: ドラッグで通過した順がそのまま発音順。下向き = ダウンストローク(低音弦から)、上向き = アップストローク(高音弦から)。途中での折り返しも通過順どおりに発音する
- **発音タイミング**: 跨いだ実時刻を保持し、サンプルオフセットへ反映(§5)。ドラッグ速度がそのままストロークの「速さ」になる
- **ベロシティ**: 跨ぎ速度(px/ms)から算出。設定: 感度、最小値 / 最大値、固定値モード
- **let ring**: ストラムで鳴った音はマウスアップ後も持続する。同じ弦が次に発音されるときに旧音を off
- **ゾーンの点灯**: 弦は発音(ピッキング)の瞬間に短くフラッシュし、即座に減衰する。発音が持続していても常時点灯はしない — 連続アップ / ダウンストロークの各ストロークが 1 回ずつ視認できるようにするため
- **点灯保持オプション**(設定、既定 OFF): 弾いた弦をストローク中ずっと点灯させ、マウスアップで全消灯する。キーボードストラムは常にフラッシュ動作
- **全消音(ミュート)**:
  - ストラムゾーン内で弦を跨がずにクリック & リリース = 全消音(ピックを当てて止めるメタファー)
  - ツールバーのミュートボタン、Esc キー
  - エディタクローズ・再生停止(panic 経路、§5)
- **キーボードストラム**(M5): Space = ダウンストローク、Shift+Space = アップストローク。弦間隔は設定値(ms)で一定速度

## 10. コードプリセット

ツールバーで **ルート(C〜B の 12)+ コードタイプ** を選ぶと、シェイプライブラリからラッチが自動配置される(コードモードが自動的に ON になる)。配置後は §8 の操作で手直しできる。

- **タイプ(v1 内蔵)**: maj / min / 7 / maj7 / m7 / sus4 / sus2 / add9 / 9 / m9 / dim / aug / 5(パワーコード)
- **フォーム切替**: 同じコードに複数のフォーム(E フォームバレー / A フォームバレー / オープン等)を持ち、「次のフォーム」ボタンで巡回する。フォームごとに押さえる位置(基準フレット)が変わる

### シェイプのデータモデル(assets/chord-shapes.json)

シェイプ = ルート弦 + 弦ごとの相対フレット(`-1` = ミュート)。ルート音から基準フレットを計算して配置する。

```json
{ "name": "E-form maj", "rootString": 6, "frets": [0, 2, 2, 1, 0, 0], "movable": true }
```

例: F メジャー = E フォームをルート F(6 弦 1 フレット)へ → 実フレット `[1, 3, 3, 2, 1, 1]`(バレーコード)。

- オープンコード(移動不可)は `movable: false` + 固定ルートで同形式に持つ(例: open C maj = x32010 → `frets: [-1, 3, 2, 0, 1, 0]`)
- シェイプは **6 弦スタンダードチューニング基準**で定義する。チューニングを変えても「同じ指の形」をそのまま適用する(出る音は変わる — 実ギターと同じ挙動)。オープンチューニング専用シェイプは将来拡張
- 7–9 弦では 6 弦シェイプを 1–6 弦に適用し、追加の低音弦はミュートとする(多弦向け拡張シェイプは将来)
- **ユーザーコード**: 現在のラッチ状態に名前を付けて保存できる(ユーザーファイル `VirtualFret/chords.json`、チューニングと同じ仕組み)— M5

## 11. UI 仕様

- **レイアウト**: 上部ツールバー / 左端ヘッド部(チューニング表示・編集) / 中央フレットボード / 右端ストラムゾーン
- **ツールバー**: 弦数、チューニングプリセット選択、コードモードトグル、コードプリセット(ルート・タイプ・フォーム)、クリア、ミュート、ベロシティ、チャンネルモード、設定(ノート名表示 / 入力ハイライト / ラッチ試聴 / 言語)
- ウィンドウはリサイズ可能。横長基調。最小サイズは意図的にかなり小さくする — Guitar Pro のフレットボード風の細い帯まで縮小でき、どこまで狭くするかはユーザーに委ねる。表示フレット数ズーム(§6)と組み合わせてコンパクト表示にできる。**起動時はこの最小サイズで開く**(広げるのはユーザーの操作)
- **デザイン**: DESIGN.md(Google Stitch で起こすデザインシステム)に従う。NoteNamer の「Ableton Live Native」デザインシステム(ダークグレー基調 + アンバー LED アクセント)の流儀を踏襲予定
- キーボード: Esc = 全消音。Space / Shift+Space = ストラム(M5)。`EDITOR_WANTS_KEYBOARD_FOCUS TRUE`

## 12. 状態保存

- **プラグインステート**(DAW プロジェクトと一緒に保存): 弦数、現在のチューニング(名前 + ノート列)、コードモード ON/OFF、ラッチ状態、ベロシティ設定、チャンネルモード、ストラム設定、表示オプション、言語。JSON 形式・バージョンフィールド付き(NoteNamer と同じ流儀)
- **ユーザーファイル**(全インスタンス共有): `tunings.json` / `chords.json`(ユーザー定義分のみ。内蔵プリセットはバイナリに内蔵)

## 15. テスト戦略

- **コアロジックの分離**: チューニングモデル、コードシェイプ解決、ストラム状態機械、ステート直列化は `juce_core` + `juce_data_structures` のみに依存させ、コンソールテスト(ctest)で回す
- 重点テスト項目:
  - シェイプ → フレット配置の解決(ルート移動、ミュート弦、指板範囲外になるケースのフォーム選択)
  - チューニング変更時の発音中ノート解放(off が「鳴らしたときの」ノート番号で出ること)
  - 同一弦モノフォニックの差し替え
  - ストラムの発音順・サンプルオフセット計算
  - ステート JSON の round-trip
- GUI は Standalone で操作検証(連打・ドラッグ往復・編集中クローズのストレス)→ 最後に必ず Live 実機
- インストールは `scripts/install-plugins.sh` 方式(`pgrep -x Live` ガード + コピー後 deep ad-hoc 再署名)

## 16. マイルストーン

- **M1**: 指板表示 + 単音クリック発音(6 弦 Standard 固定)。Live で下流のギター音源が鳴ることを確認
- **M2**: チューニング(内蔵プリセット全部 + 弦別編集 + ユーザー保存)+ 弦数 6–9 切替
